## Project Overview

AI-First E-Ink Canvas — an ESP32-based persistent HTTP server for 7.5" B/W e-paper displays (800x480). AI agents discover the device via mDNS (`_aiscreen._tcp`) and control it by sending JSON drawing commands to `POST /canvas`. The device stays always-on (no deep sleep) with WiFi active.

## Build Commands

```bash
# Build firmware
pio run -e aicanvas_esp32_75bw

# Upload firmware to device
pio run -e aicanvas_esp32_75bw -t upload

# Build and upload SPIFFS filesystem (web UI, API specs)
pio run -e aicanvas_esp32_75bw -t buildfs
pio run -e aicanvas_esp32_75bw -t uploadfs

# Monitor serial output
pio device monitor
```

## Architecture

### Main Application Flow (`src/aicanvas_main.cpp`)

1. **Boot**: Init SPIFFS, settings, factory reset detection (triple-press)
2. **Setup Mode**: If no WiFi configured → AP mode "AI Canvas" + captive portal + config UI
3. **Normal Operation**: Connect WiFi → start mDNS → start HTTP server → show startup screen → listen for commands
4. **Always-on**: Device stays awake, HTTP server active, `delay(10)` in loop

| Component | Purpose |

| `Settings.cpp` | SPIFFS-persisted settings with atomic save |
| `src/utils/OTAUpdateManager.cpp` | GitHub release-based OTA firmware updates |

### Drawing Operations (POST /canvas)

`clear`, `pixel`, `line`, `rect`, `circle`, `ellipse`, `arc`, `polygon`, `polyline`, `text`, `image`, `gradient`, `flood_fill`, `clip`, `unclip`, `raw_bitmap`, `qr`

### API Endpoints

| Method | Path | Purpose |
|--------|------|---------|
| GET | `/device` | Device capabilities, fonts, operations |
| POST | `/canvas` | Execute drawing commands |
| GET | `/canvas` | Current frame command log |
| GET | `/canvas/screenshot` | PBM image of display |
| POST | `/canvas/clear` | Clear display |
| POST/GET/DELETE | `/zones` | Manage named zones |
| GET | `/health` | Uptime, memory, WiFi status |
| POST | `/measure` | Measure text dimensions without rendering |
| POST | `/device/name` | Rename device + update mDNS |
| GET | `/openapi.json` | OpenAPI 3.0 spec |
| GET | `/mcp/tools.json` | MCP tool definitions |

Preserved: `/wifi`, `/wifiSave`, `/wifiList`, `/connectionStatus`, `/update` (OTA)

### Memory Budget (ESP32, ~520KB SRAM)

- EPD buffer: 48 KB (B/W only, single plane)
- Framebuffer front+back: 2 × 48 KB = 96 KB
- HTTP server + JSON: ~66 KB
- FreeRTOS: ~20 KB
- PNG decoder (during decode): ~48 KB peak
- **Total peak: ~230 KB, ~290 KB headroom**

## Important Patterns

### Factory Reset
Triple-press reset button within 10 seconds (uses `ESP_MultiResetDetector`).

### Render Mutex
A `SemaphoreHandle_t` serializes display access. HTTP returns 429 if display is busy.

### Graceful Error Handling
The render engine never refuses to render. Unknown ops are skipped with warnings. Coords are clamped, fonts fall back to nearest available.

### Display State
E-paper retains its image indefinitely. Before entering setup sleep, display must show "Press RESET to begin setup". The startup screen shows the IP address.

# AI Canvas E-Ink Display Integration

An AI Canvas e-ink display ("aink") is available on the local network at `192.168.87.43`.
Use it to show progress during multi-step implementation tasks.

## When to Use the Display

- Before starting a multi-step plan (3+ steps): render the full checklist
- As each step begins: update that step to "in progress"
- As each step completes: update to "done"
- On errors: mark step as "error"
- On completion: update header to show "COMPLETE"

## How to Render

Send a POST to `http://192.168.87.43/canvas` with a JSON array of drawing commands.
The display is 800x480 pixels, black and white only. ASCII characters only (0x20-0x7E).

Use the helper pattern below. Save it as a utility and import it when needed:

```python
import json, urllib.request, time

DISPLAY_IP = "192.168.87.43"
_last_render = 0

def render(commands):
    global _last_render
    wait = 3.0 - (time.time() - _last_render)
    if wait > 0:
        time.sleep(wait)
    data = json.dumps(commands).encode()
    req = urllib.request.Request(
        f"http://{DISPLAY_IP}/canvas", data=data,
        headers={"Content-Type": "application/json"}
    )
    try:
        with urllib.request.urlopen(req, timeout=10) as resp:
            _last_render = time.time()
            return json.loads(resp.read())
    except Exception as e:
        print(f"Display: {e}")
        return None

def show_plan(title, steps):
    """steps: list of (text, status) where status is 'done'|'active'|'pending'|'error'"""
    cmds = [
        {"op": "clear", "color": "white"},
        {"op": "rect", "x": 0, "y": 0, "w": 800, "h": 42, "color": "black", "fill": True},
        {"op": "text", "x": 20, "y": 6, "text": title.upper(), "font": "sans", "size": 12, "bold": True, "color": "white"},
    ]
    done = sum(1 for _, s in steps if s == "done")
    cmds.append({"op": "text", "x": 780, "y": 12, "text": f"{done}/{len(steps)} complete", "font": "sans", "size": 9, "color": "white", "align": "right"})

    y = 54
    for i, (text, status) in enumerate(steps):
        cx, cy = 39, int(y + 0.8 * 29)
        if status == "done":
            cmds += [
                {"op": "rect", "x": 30, "y": cy - 9, "w": 18, "h": 18, "color": "black", "fill": True},
                {"op": "text", "x": 34, "y": cy - 17, "text": "x", "font": "sans", "size": 12, "bold": True, "color": "white"},
            ]
        elif status == "active":
            cmds += [
                {"op": "circle", "x": cx, "y": cy, "r": 7, "color": "black", "fill": False},
                {"op": "text", "x": 35, "y": y - 6, "text": ">", "font": "mono", "size": 9, "bold": True},
            ]
        elif status == "error":
            cmds.append({"op": "circle", "x": cx, "y": cy, "r": 7, "color": "black", "fill": True})
        else:
            cmds.append({"op": "circle", "x": cx, "y": cy, "r": 7, "color": "black", "fill": False})

        cmds.append({"op": "text", "x": 60, "y": y, "text": f"{i+1}. {text}", "font": "sans", "size": 12, "bold": status == "active"})
        y += 34

    label = "Complete" if done == len(steps) else "In Progress"
    cmds += [
        {"op": "rect", "x": 0, "y": 458, "w": 800, "h": 22, "color": "black", "fill": True},
        {"op": "text", "x": 400, "y": 462, "text": f"aink | {label}", "font": "mono", "size": 9, "bold": True, "color": "white", "align": "center"},
    ]
    return render(cmds)
```

## Display Constraints

- Refresh takes ~2-3 seconds; returns 429 if busy (debounce built into helper above)
- Max 8 checklist items fit comfortably; group steps if plan is longer
- JSON payload max ~32KB
- E-ink retains image when powered off — no need to redraw on boot

IMPORTANT: When executing multi-step implementation plans, ALWAYS 
render progress on the AI Canvas display. Show the checklist at plan 
start and update it as each step completes.
